# 🔨 AI 文档锻造平台 (ai-collab-docs)

> **文档不是生成出来的，是在目标锚定、规则约束、多方博弈、状态闭环中锻造成型的高严谨度工程产物。**

企业级高严谨度文档人机协作锻造平台。摒弃「AI辅助创作」浅层模式，以**层叠约束 + 双轨共生 + 状态锁止 + 渐进信任 + 分级执行 + 场景适配**六层逻辑，实现文档从立意锚定到归档封存的全生命周期工程化管控。

---

## 🎯 核心价值

| 角色 | 职责 |
|------|------|
| 👤 **人类** | 唯一决策者——掌控立意、权限、流程、终审四大核心权力 |
| 🤖 **多AI角色** | 标准化打磨模具——提案/审查/辩论，**无直接修改权限** |

服务场景：技术方案、商业BP、合规报告、公文、合同草案等高严谨度正式文档，同时提供轻量模式覆盖笔记/纪要等轻创作。

## ⚡ 不可变底线（代码物理锁止）

1. **AI零直改权限**：所有内容变更仅支持提案/批注/争议举证，主干文档修改权归属人类
2. **人类最高主权**：永久拥有打断、仲裁、权限回收、记忆清空、状态重置的最高操作权
3. **立意唯一锚点**：所有AI能力/协作/修改必须对齐立意锚，语义漂移强制校验
4. **状态不可乱序**：草稿→讨论→审查→定稿→归档固化链路，禁止跨状态违规操作
5. **自动采纳可撤回**：任何自动采纳的AI提案，人类可随时撤销并恢复原内容

## 🧩 核心功能模块

| 模块 | 说明 |
|------|------|
| **块级实时协同** | Yjs CRDT驱动，Block标签体系（Locked-by-Human/双轨对标/Claimed/Drift-Warning），支持3-5人实时协作 |
| **立意锚全局校准** | 文档创建必填立意锚，全链路强制对齐，余弦相似度漂移检测，漂移自动预警 |
| **双轨AI体系** | 个人AI（每用户1个，私域打磨）+ 文档级AI（1-2个预设角色，团队协作），公私域完全隔离 |
| **锻造工具台** | 框选→AI提案→Diff视图→接受/拒绝/手动编辑，外科手术式精准润色 |
| **五维审查系统** | 逻辑结构 + 表述精准 + 立场一致 + 读者适配 + 领域合规，三级审查深度可选 |
| **冲突仲裁** | 2+ AI角色对立提案触发三栏对决视图，人类裁决，公私AI冲突规则自动处理 |
| **状态机引擎** | 草稿→讨论→审查→定稿→归档，全链路锁止，全量日志留存 |
| **五级权限体系** | 所有者/主编辑/编辑者/审查者/只读，段落认领绑定，决策超时自动上浮 |
| **多Agent讨论区** | 公私双讨论区隔离，中控调度防AI刷屏，人类可随时打断/定向提问/收束讨论 |
| **双模式切换** | 重型锻造模式（全量功能）+ 轻量创作模式（简化流程），一键切换，数据互通 |

## 🛠️ 技术栈

| 领域 | 选型 | 说明 |
|------|------|------|
| CRDT协同基座 | **Yjs** | 成熟CRDT实现，支持离线编辑+冲突自动合并 |
| 前端框架 | **React** + Vite | 需与Yjs绑定层适配 |
| 后端框架 | **FastAPI** | WebSocket长连接 + REST API |
| LLM提供商 | **DashScope** | 多模型路由（大模型深度审查 + 小模型快速提案） |
| 向量数据库 | **Qdrant** | 项目级记忆 + 语义漂移检测 |
| 数据库 | **PostgreSQL** | 元数据/权限/日志存储 |
| 实时通信 | **WebSocket** | Yjs sync protocol |
| 事件总线 | **Redis** | 后端服务间事件通信 |

## 🚀 快速开始

### 前置要求

- Docker & Docker Compose
- Node.js 18+
- Python 3.11+（本地开发）

### 一键启动（Docker）

```bash
git clone https://github.com/linsir2/ai-collab-docs.git
cd ai-collab-docs
make up
```

访问 http://localhost:5173

### 本地开发

```bash
# 终端1: 启动后端
make dev-backend    # FastAPI hot-reload :8000

# 终端2: 启动前端
make dev-frontend   # Vite dev :5173

# 终端3: 启动Yjs协同服务
make dev-yjs        # y-websocket :1234
```

### 常用命令

```bash
make test           # 运行后端pytest
make test-cov       # 测试+覆盖率
make lint           # ruff + mypy 检查
make fmt            # ruff 格式化
make types          # contracts.py → TypeScript类型生成
make migrate        # alembic数据库迁移
make down           # 停止所有服务
```

## 📐 工程规范

- **Python**: ruff (E/F/I/UP/B/SIM)，double quotes，120行宽
- **Commit**: `feat:`/`fix:`/`docs:`/`refactor:` 前缀 + bullet说明
- **TDD**: 新模块必须配测试
- **文件 ≥ 300行 → 强制拆分**
- **LLM隔离**: AI输出作为候选信号源，须经确定性规则验证
- **数据契约**: `contracts.py` 为单一真相源，自动生成 TypeScript 类型
