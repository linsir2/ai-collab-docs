# AI文档锻造平台 产品需求文档（PRD）

## 一、核心设计哲学（纲领性唯一准则）
摒弃行业通用的「AI辅助创作」浅层模式，定义产品核心逻辑：**文档不是生成出来的，是在目标锚定、规则约束、多方博弈、状态闭环中锻造成型的高严谨度工程产物**。

构建全新人机协作范式，形成六层底层逻辑：
1. **层叠约束**：从数据层、规则层、AI能力层、交互层形成自上而下的嵌套式约束，所有AI行为、协作动作、文档修改均无突破空间
2. **双轨共生**：公私双轨、人机双轨、新旧内容双轨、轻重模式双轨并行，兼顾创作灵活性与结果严谨性
3. **状态锁止**：文档全生命周期状态可定义、可锁止、可流转、可回溯，杜绝无边界修改、无限拉扯、版本混乱
4. **渐进信任**：AI权限与能力随历史表现动态调整，从完全人工审批到有条件自动采纳，平衡摩擦成本与安全性
5. **分级执行**：AI任务按优先级与成本分级，高价值任务全量执行，低价值任务自动降级，优化资源与体验
6. **场景适配**：针对不同文档类型与用户群体提供差异化模式，避免重型框架成为轻量创作的阻力

**核心权责关系**：人类为文档唯一所有权决策者，掌控立意、权限、流程、终审四大核心权力；多AI角色为**有立场、有记忆、有约束、可追溯**的标准化打磨模具，仅通过施压、审查、辩论、提案完成内容优化，无自主修改权限。

**不可逾越底线规则**：
1. AI零直改权限：所有内容变更仅支持提案、批注、争议举证，主干文档修改权完全归属人类，底层网关硬锁止
2. 人类最高主权：永久拥有打断、仲裁、权限回收、记忆清空、状态重置的最高操作权
3. 立意唯一锚点：所有AI能力、协作讨论、内容修改必须对齐立意锚，语义漂移强制校验，冲突无条件适配核心立意
4. 状态不可乱序：文档流转状态锁止，禁止跨状态违规操作，所有变更留存全量日志
5. 自动采纳可撤回：任何自动采纳的AI提案，人类可随时撤销并恢复原内容，不影响历史记录

**核心市场定位**：专注服务企业级高严谨度正式文档创作场景（技术方案、商业BP、合规报告、公文、合同草案），同时提供轻量模式满足多场景需求。

## 二、整体四层架构（层叠约束闭环架构）
采用层叠约束闭环架构，四层架构自下而上层层赋能、层层校验、层层约束，每一层能力均被下一层底座兜底、上一层场景落地，彻底解决传统AI文档工具「能力松散、规则失效、权限失控」问题。

### 1. 底层协同数据层（约束底座）
为全平台提供强一致、可追溯、可标记、可权限拦截的核心数据地基，包含**CRDT块级文档结构、立意锚元数据体系、全量版本快照、操作增量日志、权限标签体系**，所有上层功能均基于该层规则运行，无例外绕过入口。

### 2. 业务规则引擎层（约束中枢）
整合六大核心引擎，形成统一规则中枢，实现全链路自动化管控：**文档状态机引擎、立意校准引擎、人机审批仲裁引擎、AI权限安全网关、渐进信任引擎、预算控制引擎**，所有AI行为、用户操作、状态流转均经过引擎校验后执行，实现规则硬落地。

### 3. AI能力层（约束执行层）
构建双轨AI体系，依托项目级向量记忆、多Agent中控调度、五维审查模型，在规则约束下输出标准化、立场统一、无漂移的AI能力，杜绝随机输出、逻辑混乱、重复拉扯问题。配置分级执行调度器与成本监控模块，实现AI资源的精细化管理。

### 4. 前端交互层（场景落地层）
统一锻造工具交互体系，整合精细化润色、争议仲裁、状态提示、审批闭环、多角色协作视图，将底层约束规则可视化、可操作化，让用户直观感知人机权责边界与锻造流程。提供双模式切换入口与进度可视化面板，提升用户体验。

## 三、核心模块设计
### 模块一：块级实时协同数据模块（标签体系标准化+溯源全覆盖）
**核心定位**：全平台权限、约束、状态、溯源的最小执行单元，单Block承载完整的锻造属性。

**核心特性**：
1. 标准化块级标签体系：统一四类核心标签的底层执行逻辑，`Locked-by-Human`黑区硬拦截、`[双轨对标]`版本留存、`Claimed-by-XXX`权责绑定、`Drift-Warning`漂移预警，所有标签由数据层管控，前端无法篡改
2. 协同区分强化：精准区分人类光标、AI虚拟光标、操作状态，多Block并行独立操作互不干扰，适配大规模团队协作
3. 快照溯源能力：支持审查态自动快照与用户手动快照双模式，提供快照备注、变更原因留存功能，支持任意版本逐块差异对比、一键回滚，增量日志精确到单字符变更
4. 离线协同支持：完善离线编辑冲突合并策略，元数据与内容同步落盘，多端上线后自动对齐所有状态与标签

### 模块二：立意锚全局校准系统（全链路强制锁止+量化迭代）
**核心定位**：平台唯一全局目标基准，所有业务行为的最高校验标准，解决团队协作目标分歧、内容跑偏、风格割裂核心痛点。

**核心特性**：
1. 数据结构强制绑定：CRDT根节点固化Anchor字段，文档创建必填、不可删除，修改全程留痕、版本可追溯，杜绝无立意创作
2. 全链路强制校准：所有AI调用请求头部强制注入立意锚，系统前置拦截偏离内容，大模型输出无权限绕过校准规则
3. 多维度语义漂移检测：保留多场景阈值差异化配置，采用上下文感知校验逻辑，结合向量模长、维度方差等特征辅助决策，引入历史数据自学习机制动态微调阈值，仅在审查态进行增量检测
4. 漂移处理闭环：系统显示黄色预警提示并附带判断依据，提供「内容修正、立意迭代、忽略本次」三合规路径，所有调整记录存入项目记忆，持续优化后续创作校准精度

### 模块三：核心锻造工具集（精细化可控+全流程闭环）
**核心定位**：人机锻造的核心交互载体，实现「精准、可控、可审、可追溯、低摩擦」的精细化文档打磨，杜绝粗放式AI生成。

#### 3.1 外科手术式精准润色台
采用框选锁定逻辑，严格限定AI修改边界，仅在用户指定Block范围内生成提案；支持自定义上下文适配长短文本创作；配置连贯性后置校验机制，自动排查逻辑、语气、风格断裂问题；输出标准化Diff视图+校验报告+角色溯源信息，修改效果直观可控。

#### 3.2 四维审批闭环体系
基于双轨共生思路设计审批逻辑，形成完整人机决策闭环，提供五种标准化操作选项：全盘合并、双轨留存、拒绝批注、驳回重造、手动编辑。所有审批行为同步更新AI项目记忆，实现AI能力持续迭代，双轨保留模式解决「新版优化、旧版参考」的创作矛盾。

**渐进式信任与自动采纳机制**：
1. AI角色信任分体系：为每个AI角色维护项目级信任分，初始值60分，根据提案审批结果动态调整
2. 信任分与权限映射：60-79分所有提案必须人工审批；80-89分单句修改提案可自动采纳；90-100分段落级修改提案可自动采纳
3. 自动采纳安全机制：所有自动采纳提案显示绿色标识，支持一键撤销，生成独立历史记录，黑区与公共区Block默认关闭自动采纳
4. 低摩擦快速通道：提供"一键接受所有低风险修改"按钮，支持批量审批，预设常用拒绝理由模板

#### 3.3 多AI角色冲突仲裁台
精准识别多AI对立提案，采用三栏式对决视图标准化展示争议双方立场、论据、方案与核心分歧；人类仲裁结果实时落地文档，同时固化为项目级立场记忆，从根源避免同类争议重复发生。
1. **冲突角色范围**：冲突判定覆盖文档级AI、个人AI两类角色，满足「2个及以上角色针对同一Block产出对立提案/审查意见」即触发冲突仲裁。
2. **纯个人AI冲突规则**
   - 多个个人AI产生冲突时，冲突视图仅在当前用户的个人讨论区展示，其他成员不可见；
   - 裁决权限仅归属当前用户，裁决结果存入**个人AI项目记忆**，不影响文档公共记忆；
   - 申诉、防拉扯机制与全局规则保持一致，申诉内容仅私域可见。
3. **个人AI与文档级AI混合冲突规则**
   - 冲突视图对全体文档成员可见，区分标注【个人AI】/【文档级AI】角色标签；
   - 冲突优先由段落认领人（个人AI所属用户）处理，超时后按照动态决策上浮链逐级流转；
   - 裁决结果双向存储：同步至文档全局项目记忆、对应用户个人AI记忆，实现公私记忆隔离。
4. **功能禁用场景**
   - 轻量创作模式下，全局关闭冲突检测与仲裁能力；
   - 标记`Locked-by-Human`的AI黑区、仅单个AI角色工作、纯闲聊类讨论，均不触发冲突检测；
   - 离线编辑期间暂停冲突检测，联网同步完成后批量校验。

### 模块四：AI身份与权限管网（双轨共生体系标准化落地）
**核心定位**：构建公私分离、权责清晰、边界绝对锁止的AI协作体系，解决多人协作AI干扰、记忆污染、权限混乱、成本失控问题。

#### 4.1 个人AI+文档级AI双轨体系
标准化双轨AI的归属、可见性、触发时机、审批权限、记忆范围、使用场景，实现私人打磨与公共协作完全隔离：个人AI服务用户私域创作，全程私密；文档级AI服务团队公共评审，全员可见、流程公开，双轨并行互不干扰，覆盖全创作场景。

#### 4.2 项目级隔离记忆机制
记忆严格限定项目周期，归档项目记忆自动封存，杜绝跨项目记忆污染；记忆覆盖角色人设、审批结果、仲裁结论、人类反馈全维度；支持手动单条删除、批量清空，记忆检索精准匹配当前文档场景，AI输出持续贴合团队创作风格与立场。

#### 4.3 黑区硬保护机制
底层网关物理拦截AI所有编辑、批注权限，锁定区块完全杜绝AI越权操作；仅指定权限用户可修改锁定状态，可视化锁形标识直观展示权限边界。

#### 4.4 AI全局可控机制
提供三档全局触发模式（仅@触发、阶段触发、全程监听），支持单角色静默、单能力开关精细化管控，用户可按需灵活控制AI参与度。

#### 4.5 AI预算与成本控制体系
1. 三级预算防御体系：设置单次任务、单文档、团队月度三级预算上限
2. 分级执行策略：根据任务优先级匹配不同能力的模型与检查项
3. 用户可感知进度反馈：显示预计耗时、预算消耗与实时进度，预算使用超过80%时发出预警
4. 降级与熔断机制：预算不足时自动降级为小模型执行，连续调用失败时触发熔断

### 模块五：文档状态流转引擎（状态锁止+防拉扯闭环）
**核心定位**：以状态锁止为核心，标准化文档全生命周期，规范所有人机操作行为，杜绝流程混乱、无限修改、无效拉扯。

**核心特性**：
1. 固化唯一流转链路：草稿态→讨论态→审查态→定稿态→归档态，仅保留合规回退路径，禁止跨状态违规操作
2. 各状态权责细分：明确不同状态下人类、AI的权限边界与行为规则，状态切换自动触发对应能力启停、快照生成、校验开启
3. 基于论据质量的防拉扯机制：审查态AI对驳回的批注可进行1轮结构化申诉，申诉必须包含核心论点、证据支撑与修改建议；系统自动对申诉论据质量评分，≥7分推送给人类重新审核，<7分自动关闭争议

### 模块六：多人团队协作体系（权责下沉+决策高效）
**核心定位**：适配企业级正式团队协作，实现「人人有责、层层审批、超时可控、分歧可裁」的标准化协作模式。

**核心特性**：
1. 五级人类权限体系：细化所有者、主编辑、编辑者、审查者、只读用户的全量操作权限，权责清晰无重叠
2. 段落认领机制：区块权责绑定个人，专属区块优先审批、公共区块统一管控
3. 动态决策上浮链：支持按文档类型配置不同超时规则（快速迭代项目4小时、常规项目24小时、正式公文72小时）；超时后逐级发送提醒并上浮审批，原审批人仍可继续审批，最终结果以最晚审批为准；支持委托审批功能，所有操作记录全量留存

### 模块七：多Agent协作讨论区（可控发散+高效收敛）
**核心定位**：将AI无序发散思维，转化为可控、有序、可落地、可沉淀的团队头脑风暴能力。

**核心特性**：
1. 公私双讨论区隔离：公共讨论服务团队协作、个人讨论服务私域思考，场景完全拆分
2. 中控调度能力：限制AI连续发言、并发调用数量，智能分配发言权重，防止AI刷屏、单一角色主导讨论
3. 人类绝对控制权：支持随时打断、定向提问、一键收束讨论，讨论内容可快速沉淀为文档正式内容

### 模块八：个性化配置与企业管理
**核心特性**：
1. 自定义AI角色卡：支持创建无限个自定义AI角色，配置角色名称、头像、核心立场、能力标签、触发方式与活跃度；提供官方预设角色库，支持团队统一角色模板复用
2. 模板中心：提供多种预设文档模板，固化立意锚、检测阈值、审查标准、适配角色与超时规则；支持用户保存自定义模板并分享给团队成员
3. 企业团队管理：支持创建团队、邀请成员，提供团队默认规则配置、权限批量配置、文档进度大盘、操作日志审计、预算管理面板功能

### 模块九：五维全量审查系统（标准化+可量化+分级执行）
固化**逻辑结构、表述精准、立场一致、读者适配、领域合规**五大核心审查维度，实现文档质量全维度标准化校验；审查结果分类展示、可筛选、可追溯，所有问题批注统一进入审批闭环。

提供三级审查级别：快速审查（仅执行表述精准与立场一致性检查）、标准审查（执行所有核心检查项）、深度审查（全量执行所有检查项，包含外部资料检索与交叉验证），用户可根据文档重要性选择。

### 模块十：双模式切换系统
针对不同文档类型与用户需求，提供差异化工作模式，避免重型锻造框架成为轻量创作的阻力。

#### 10.1 重型锻造模式（默认）
- 适用场景：技术方案、商业BP、合规报告、公文、合同草案等高严谨度正式文档
- 核心特性：全量启用所有锻造功能、完整状态流转、严格权限管控、多AI角色对抗、全量审查与校验
- 设计目标：追求文档质量最大化

#### 10.2 轻量创作模式
- 适用场景：个人笔记、会议纪要、创意文案、邮件草稿等轻量文档
- 核心特性：简化状态流转（仅保留草稿态与定稿态）、关闭自动审查与语义漂移检测、简化审批流程、仅启用个人AI、关闭段落认领与决策上浮链
- 设计目标：追求创作效率最大化

支持两种模式间自由切换，切换时系统自动提示模式差异与影响，文档数据完全互通。

## 四、企业级落地支撑能力
为保障产品在企业环境中的稳定运行与合规使用，提供以下核心支撑能力：
1. **算力成本优化机制**：包含AI调用节流、记忆缓存、无效校验降级、并发限流策略，大幅降低多Agent、全量审查的Token消耗
2. **企业合规审计体系**：全操作留痕、权限变更日志、审批溯源记录、归档合规备份，满足政企审计规范
3. **精细化自定义配置**：支持团队批量配置漂移阈值、审批超时时长、AI活跃度、状态流转规则、预算上限，适配不同企业流程规范
4. **文档质量复盘模块**：自动生成锻造前后质量评分、问题整改统计、优化维度复盘报告，实现文档创作可量化迭代

## 五、核心创新壁垒
1. **层叠式硬约束壁垒**：所有规则下沉数据层、网关层，形成多层嵌套约束，无任何绕过漏洞，彻底守住人类主导权
2. **立意锚全局校准壁垒**：将抽象写作目标转化为可量化、可校验、可追溯的系统能力，解决行业创作跑偏核心痛点
3. **双轨AI协作壁垒**：公私AI体系完全隔离，兼顾个人打磨灵活性与团队协作规范性，填补行业空白
4. **状态锁止工程化壁垒**：将文档创作升级为全生命周期标准化工程，终结团队文档版本混乱、改无止境问题
5. **精细化人机权责壁垒**：四维审批、仲裁闭环、决策上浮，将模糊的人机协作关系拆解为可落地的标准化流程
6. **项目级长效记忆壁垒**：AI拥有持续统一的项目立场，告别传统AI临时对话、无记忆、无立场的工具属性
7. **渐进式信任与成本控制壁垒**：动态调整AI权限与资源分配，在保证人类主导权的前提下，平衡摩擦成本与算力消耗

## 六、全链路业务流转
### 重型锻造模式全链路
文档创建（锚定立意+配置权限+选择模式）→ 草稿态（个人AI私密打磨+段落认领+渐进式自动采纳）→ 讨论态（团队+文档AI公开头脑风暴+内容沉淀）→ 审查态（自动快照+分级审查+漂移检测+提案审批+争议仲裁+预算控制）→ 定稿态（只读锁止+权限回溯管控）→ 归档态（全量封存+合规备份+记忆冻结）

### 轻量创作模式全链路
文档创建（选择轻量模式）→ 草稿态（个人AI辅助创作+一键接受/拒绝提案）→ 定稿态（只读锁止）→ 归档态（全量封存）

全流程状态锁止、规则约束、日志留存，完美适配从个人轻量创作到企业级正式文档锻造的全场景需求。
