### 0.3 桌面端特有设计原则

1. **窗口弹性替代响应式**：无移动端断点。窗口最小尺寸 1024×680，低于此限制时三栏等比例收缩而非折叠
2. **原生Chrome优先**：标题栏使用 Tauri 原生窗口装饰（macOS 红绿灯 / Windows 控制按钮），不自定义窗口控件
3. **快捷键为第一操作**：所有高频操作绑全局快捷键，菜单栏为快捷键的可见入口，工具栏为鼠标兜底
4. **面板可弹出**：右栏任意面板可右键→"弹出为独立窗口"，桌面端多显示器场景释放生产力
5. **文件即文档**：`.forge` 文件关联双击打开，应用内 `Ctrl+O` 文件选择器打开，文件系统为第一存储（同步到后端为备份）

```

│  │ user@company.com           │ │
│  └────────────────────────────┘ │
│                                  │
│  密码                            │
│  ┌────────────────────────────┐ │
│  │ ●●●●●●●●                  │ │

### 3.2 左栏：校准中枢（300px，始终可见，无折叠）

**立意锚卡片**（始终可见，`Ctrl+1` 聚焦）：
- `--bg-elevated` 底，左边框 2px `--accent`
- 标题 "立意锚" 12px 大写 `--text-muted`，正文 14px `--text-primary`
- 右下角 `[查看历史]` → 立意迭代时间线独立窗口

**语义漂移检测仪 (VU Meter)**：
- 当前余弦相似度：大号等宽数字 `0.87`（JetBrains Mono）
- 6段式跳变条：>0.85 绿色，0.8-0.85 橙色，<0.8 红色
- 漂移速率：`连续3次<0.85，当前第2次` / `WARN 速率预警`

**文档结构树**：
- 缩进树，每行 28px，Block编号等宽灰色小字
- 当前Block高亮 `--accent-subtle`，状态图标前置（LOCK/WARN/CLAIM/CHECK）
- 支持拖拽重排Block顺序（Tauri原生拖拽事件）

### 3.3 中栏：块级编辑器（桌面端优化）

**编辑器特性**：
- 纯黑 `--bg-page` 底，无边界面布
- Block 默认：`--bg-surface` 底，左边框 2px `--border-subtle`
- **焦点追踪**：当前Block左边框变 `--accent` 3px（桌面端加粗），非焦点Block淡化至 60% opacity
- **虚拟滚动**：仅渲染可视区+上下各10个Block（桌面端内存充裕，预渲染窗口加大）
- **原生滚动条**：使用系统滚动条样式（Tauri WebView继承系统主题）
- **原生右键菜单**：通过 Tauri IPC 触发系统上下文菜单（复制/粘贴/全选/撤销/重做），而非自定义DOM菜单

**Block 状态视觉标识**：

| Block 状态 | 左边框 | 右上角标签 |
|-----------|--------|-----------|
| 正常可编辑 | `--border-subtle` 2px | 无 |
| 人类锁定 | `--danger` 3px 实线 | `LOCK Locked-by-Human` |
| 漂移警告 | `--warning` 2px 虚线（脉冲动画） | `WARN Drift-Warning` |
| 双轨留存 | `--accent` 2px 虚线 | `[双轨对标]` |
| 段落认领 | `--info` 2px 点线 | `Claimed-by-张三` |
| 自动采纳(v1) | `--success` 2px 发光 | `[Auto-Accepted]` |

**框选锻造触发**：
- 鼠标拖选文本 → 工具条浮于选区上方（含「AI提案」「AI审查」「讨论」按钮）
- 快捷键：`Ctrl+Shift+F` 对选中文本触发锻造
- 框选浮层在300ms后淡入，鼠标移出1s后淡出

### 3.4 右栏：双轨AI交互中心（420px）

**Tab切换**：`[私域打磨 My AI]` | `[公域博弈 Team Agents]` （`Ctrl+2/Ctrl+3` 切换）

**私域打磨面板**：
- 个人AI角色头像+名称+信任分
- 提案列表：每条提案含 Diff 概览（+N/-M行）、状态、时间、[采纳][拒绝][展开] 按钮
- 工具条：[@触发AI提案] [手动请求审查] [记忆面板]

**公域博弈面板**：
- 文档级AI角色列表（1~2个预设角色卡片）
- 每个角色的提案独立排列
- 提案溢出警告：80%橙色/95%弹窗/100%阻断

**面板弹出**：右栏顶部 `[↗]` 按钮 → 通过 Tauri IPC `invoke('popout_panel', {panel: 'ai_forge'})` 弹出为独立窗口

### 17.1 E-STOP 三级兜底逻辑

熔断触发后，Rust `ai_scheduler::circuit_breaker` 自动执行（<300ms），UI 显示「正在安全保存...」遮罩：

| 级别 | 对象 | 动作 | 恢复方式 |
|------|------|------|---------|
| **L1** | 正在执行的 AI 任务 | 发送中断信号，标记 `INTERRUPTED`，已生成的部分提案保留为 `DRAFT`（带 E-STOP 时间戳），不注入文档 | 恢复后从"熔断恢复"对话框手动恢复 |
| **L2** | 未保存提案 + 离线数据 | SQLite 立即 flush：Yjs update 当前快照、未提交提案池、离线操作队列、冲突仲裁草稿。写入后校验 checksum | 启动时自动检测 checksum，损坏则提示从云端恢复 |
| **L3** | 未归档冲突 | 所有处于仲裁态的冲突条目强制标记 `E_STOPPED`，冻结不丢弃 | 恢复后逐条进入仲裁台重新裁决 |

### 17.2 熔断恢复对话框

应用重启后若检测到 E-STOP 标记存在，弹出模态框（520×480px）：

```

## 三十四、Tauri 架构集成要点 (Tauri Integration Notes)

### 26.1 IPC 通信模型

```
┌──────────────────────────────────────────────┐
│  React WebView（渲染进程）                     │
│                                               │
│  import { invoke } from '@tauri-apps/api/core';│
│                                               │
│  // 文件操作                                  │
│  invoke('open_file_dialog') → 文件内容         │
│  invoke('save_file', {path, content})         │
│                                               │
│  // 窗口管理                                  │
│  invoke('create_window', {label, url, ...})   │
│  invoke('focus_panel', {target})              │
│                                               │
│  // 系统集成                                  │
│  invoke('show_notification', {title, body})   │
│  invoke('get_recent_files') → 文件列表         │
│                                               │
│  // 离线                                      │
│  invoke('queue_offline_edit', {op, data})     │
│  invoke('get_sync_status') → 待同步计数        │
└──────────────┬───────────────────────────────┘
               │ Tauri IPC (JSON/serde)
┌──────────────▼───────────────────────────────┐
│  Rust Core（主进程）                           │
│                                               │
│  #[tauri::command]                            │
│  fn open_file_dialog() → Result<String>       │
│  fn save_file(path, content) → Result<()>    │
│  fn create_window(label, url, w, h) → Result │
│  ...                                          │
│                                               │
│  插件: tauri-plugin-fs / tray / shortcut /    │
│        notification / window-state / sql      │
└──────────────────────────────────────────────┘
```

### 26.2 安全约束

- Tauri CSP（内容安全策略）：仅允许 WebView 连接到 `localhost:8000`（API）和 `localhost:1234`（Yjs），禁止任意出站HTTP
- 文件系统访问：Rust 侧限制在 `~/ForgeDocs/` 目录沙箱内，IPC命令校验路径不越界
- IPC 验证：所有 `invoke` 命令在 Rust 侧验证参数类型和权限，禁止 `.unwrap()` 崩溃
- 凭据存储：使用系统 Keychain（macOS）/ Credential Manager（Windows）/ Secret Service（Linux），不落地明文

### 26.3 构建与打包

```bash
# 开发模式（React hot-reload + Rust编译）
cd services/desktop && cargo tauri dev

# 生产构建
make build-desktop
# → src-tauri/target/release/bundle/
#   macOS:   Forge_0.1.0_x64.dmg (~12MB)
#   Windows: Forge_0.1.0_x64.msi (~10MB)
#   Linux:   Forge_0.1.0_amd64.deb (~8MB)
```

## 三十五、设计决策记录

| 决策 | 选择 | 原因 | 权衡 |
|------|------|------|------|
| 平台 | Tauri v2（非Electron） | 包体积~5MB、内存~50MB起步 | Rust开发门槛较高 |
| 风格 | Dark Mode (OLED) | 减少眼疲劳；契合"锻造"工业感 | 不适合明亮环境 |
| 主色 | #3eb8b0 冷青 | 淬火/锻造意象；区别于科技蓝 | 非传统办公软件色 |
| 布局 | 300:flex:420 三栏 | 桌面端宽屏优化，黄金比例 | 小屏(<1024px)体验下降 |
| 虚拟滚动 | 可视区+上下10 Block | 桌面内存充裕，预渲染窗口加大 | 比Web端多消耗内存 |
| 右键菜单 | 系统原生 | 遵循桌面交互惯例 | 与Web端自定义菜单不同 |
| 字体 | Inter+JetBrains Mono | 桌面端系统字体优先 | 非系统默认中文字体 |
| 圆角 | 2/4/8px | 锐利直角强化工业严谨 | 温暖感不足 |
| 窗口弹性 | 替代响应式 | 桌面端无需移动端适配 | 极小窗口体验降级 |
| 原生通知 | tauri-plugin-notification | 系统级，无需应用在前台 | macOS需权限授权 |
| 离线存储 | Rust侧SQLite | 比浏览器IndexedDB更可靠 | 需Rust侧SQL维护 |

## 三十六、可拓展性声明

### 28.1 分层架构保证

```
┌──────────────────────────────────────────┐
│  界面视图层 (Views)                       │  ← v2 新增页面/面板/模态
│  本文全部28节界面属于此层                  │     无破坏性
├──────────────────────────────────────────┤
│  复合组件库 (Composite Components)        │  ← v2 新增审批流/仲裁维度
│  锻造台Diff / 仲裁台 / 审批视图 / 讨论区   │     扩展现有组件props
├──────────────────────────────────────────┤
│  原子组件库 (Atomic Components)           │  ← v2 新增按钮变体/标签颜色
│  Button(5级) / Tag(5种) / Modal(2档)      │     追加variant即可
├──────────────────────────────────────────┤
│  设计Token层 (Design Tokens)              │  ← v2 新增主题/品牌色
│  CSS变量 + 间距 + 圆角 + 字体              │     变量热替换
├──────────────────────────────────────────┤
│  Tauri IPC 桥接层 (IPC Bridge)            │  ← v2 新增原生能力
│  Rust commands + 插件                      │     追加#[tauri::command]
└──────────────────────────────────────────┘
```

### 28.2 不可破坏的契约

- **Dark Mode OLED 基调**：所有新增界面必须以 `--bg-page: #0d0d0f` 为根底色
- **三栏黄金比例**：新增面板优先放入右侧420px栏，不改变 300:flex:420 比例
- **CSS变量引用强制**：新增样式禁止硬编码颜色/间距/圆角
- **组件变体追加不改签**：新增按钮variant只能追加，不能修改前5个的样式语义
- **状态流转只追加**：新增状态只能追加在"归档态"之后
- **Tauri IPC为唯一原生通道**：不绕过IPC直接访问系统API
- **所有HTTP/WS走后端**：桌面端不直连第三方服务（LLM除外），统一走后端代理

## 二十九、全局视图切换器 (Global View Switcher)

> 依据: problem.md §三 | 对应 终版审查整改 — 三级视图隔离
> 状态: MVP强制落地

### 29.1 视图切换规则（由账号全局身份锁定）

| 账号全局身份 | 可切换视图 | 默认视图 |
|-------------|-----------|---------|
| 普通个人用户 | 仅【创作视图】 | 创作视图 |
| 团队管理员 | 【创作视图】↔【团队管理视图】 | 创作视图 |
| 运维技术账号 | 仅【运维监控视图】（需二次密码校验） | 运维视图（校验后） |

切换入口：顶部工具栏最左侧，图标+下拉选择器，全局普通用户不可点击团队管理选项（置灰+tooltip说明权限边界）。

### 29.2 视图切换时的状态保持规则（problem.md §三审查意见#4）

- 同窗口切换：编辑器实例**挂起而非销毁**，保留光标位置、未保存草稿、AI讨论区状态
- 独立窗口切换（Tauri多窗口）：两个视图各自独立窗口，通过Rust侧`ipc`模块同步关键状态（预算上限变更、团队成员调整等）
- 切换回创作视图时：从挂起状态恢复，Yjs重新连接房间，离线的Block编辑自动合并
- 团队管理视图改预算上限 → 创作视图预算面板实时刷新（通过Redis pub/sub事件总线通知）

### 29.3 运维二次校验规则（problem.md §三审查意见#5）

- 运维账号登录后**默认停留在创作视图（只读）**
- 切换到运维视图时弹窗要求**二次密码校验**（非JWT token，需重新输入密码）
- 校验通过后会话有效期：**30分钟无操作自动锁定**，需重新校验
- 运维视图激活期间：文档协同WebSocket连接**降级为只读presence**（不发送编辑事件，不参与Awareness广播），防止运维操作干扰协作者
- 锁定后自动切回创作视图（只读），未经二次校验不可再切

## 三十五、三层权限视图切换时的跨视图状态同步

> 依据: problem.md §三审查意见#4 | MVP强制落地

```
创作视图 ←→ 团队管理视图 切换时：
  创作视图:
    - 编辑器实例挂起（光标/滚动/选区保留）
    - Yjs连接保持（离线queue继续收集编辑）
    - AI讨论区状态冻结
  团队管理视图:
    - 加载团队成员列表/预算面板/规则配置

切换回创作视图:
    - 恢复编辑器实例
    - 检查期间是否收到角色变更事件 → 如有则弹出toast通知
    - 离线编辑自动Yjs merge
```
