### 0.3 排版铁律

- h1 → 页面标题，1.5rem，weight 600，每页仅一个
- h2 → 区块标题，1.25rem，weight 600
- h3 → 子区块标题，1.125rem，weight 600
- body → 正文，0.875rem，line-height 1.65
- small → 辅助/元数据，0.8125rem，`--text-muted`
- code/status/数值 → JetBrains Mono，0.8125rem，tabular-nums

### 0.4 按钮层级

| 级别 | 用途 | 样式 |
|------|------|------|
| Primary | 主操作 | `--accent` 底，白字，36px高，左右16px padding |
| Secondary | 次要操作 | 透明底，`--border-default` 1px边框，36px高 |
| Tertiary | 文字操作 | 无边框无底，`--text-secondary`，hover变`--accent` |
| Danger | 破坏性操作 | `--danger-bg` 底，`--danger` 文字+border |

状态覆盖: default / hover(亮度+8%) / active(亮度-4%) / focus(2px outline) / disabled(0.35) / loading

### 3.2 左栏：校准中枢

**立意锚卡片**（始终可见）：
- `--bg-elevated` 底，左边框 2px `--accent`
- 标题 "立意锚" 12px 大写，正文 14px
- 右下角 `[查看历史]` → 立意迭代时间线抽屉

**语义漂移检测仪 (VU Meter)**：
- 当前余弦相似度：大号等宽数字 `0.87`
- 6段式跳变条：>0.8 绿色，0.8-0.85 橙段，<0.8 红色段
- 漂移速率：`连续3次<0.85，第2次` / `WARN 速率预警`

**文档结构树**：
- 章节缩进树，每行 28px
- Block编号等宽灰色小字
- 当前Block高亮 `--accent-subtle`
- 状态图标前置（LOCK/WARN/CHECK）

### 3.3 中栏：块级编辑器

**编辑器特性**：
- 纯黑 `--bg-page` 底，无边界画布
- Block 默认：`--bg-surface` 底，左边框 2px `--border-subtle`
- 焦点追踪：当前Block左边框变 `--accent`，非焦点Block淡化至 70% opacity
- 虚拟滚动：仅渲染可视区+上下各5个Block

**Block 状态视觉标识**：

| Block 状态 | 左边框 | 右上角标签 |
|-----------|--------|-----------|
| 正常可编辑 | `--border-subtle` 2px | 无 |
| 人类锁定 | `--danger` 实线 | `LOCK Locked-by-Human` |
| 漂移警告 | `--warning` 虚线 | `WARN Drift-Warning` |
| 双轨留存 | `--accent` 虚线 | `[双轨对标]` |
| 段落认领 | `--info` 点线 | `Claimed-by-张三` |
| 自动采纳(v1) | `--success` 发光 | `[Auto-Accepted]` |

### 3.4 右栏：双轨AI交互中心

**Tab切换**：`[私域打磨 My AI]` | `[公域博弈 Team Agents]`

**私域打磨面板**：
- 个人AI角色头像+名称+信任分
- 提案列表：编号/Block/时间/变更摘要/操作按钮
- 私有提案计数：`私有池 127/400`
- 输入框支持 @提及

**公域博弈面板**：
- 文档级AI角色列表（1~2个预设）
- 提案列表同上，显示角色来源
- 公共提案计数：`公共池 643/800`
- 多AI对立提案 → 触发冲突仲裁横幅

## 二十二、设计可拓展性声明 (Extensibility by Design)

### 22.1 分层架构保证

本UI设计采用**四层解耦架构**，每层独立演进，互不锁死：

```
┌──────────────────────────────────────────┐
│  界面视图层 (Views)                       │  ← v2 新增页面/面板/模态
│  本节全部21节界面属于此层                  │     无破坏性
├──────────────────────────────────────────┤
│  复合组件库 (Composite Components)        │  ← v2 新增审批流步骤/仲裁维度
│  润色台Diff / 仲裁台 / 审批视图 / 讨论区   │     扩展现有组件props
├──────────────────────────────────────────┤
│  原子组件库 (Atomic Components)           │  ← v2 新增按钮变体/新标签颜色
│  Button(4级) / Tag(5种) / Modal(2档)      │     追加variant即可
├──────────────────────────────────────────┤
│  设计Token层 (Design Tokens)              │  ← v2 新增主题/品牌色切换
│  CSS变量 + 间距尺度 + 圆角 + 字体          │     变量热替换，零代码改
└──────────────────────────────────────────┘
```

### 22.2 v1/v2 扩展场景演练

| 扩展需求 | 对应PRD模块 | 操作方式 | 前端改动量 |
|---------|-----------|---------|-----------|
| 五维审查→完整5维(v1) | 模块九 | 审查结果视图增加3个维度行，复用现有进度条+问题列表组件 | ~20行 |
| 自动采纳(v1) | 模块三/四 | 第19节隐藏CSS移除，信任分≥80触发Auto-Accepted标签+撤销按钮 | ~15行 |
| 双轨留存/驳回重造(v1) | 模块三 | 审批按钮解锁第4、5个按钮，状态机增加对应流转路径 | ~10行 |
| 文档质量复盘面板 | PRD §四.4 | 三栏右栏新增[质量复盘] Tab，复用评分卡片+统计表格 | ~40行 |
| 决策上浮链可视化(v1) | 模块六.3 | 仲裁台底部增加超时升级时间线，复用现有时间线组件 | ~25行 |
| 外部平台导入(飞书/语雀) | 场景 | Pipeline Dashboard增加"导入"入口，上传模态+解析进度条 | ~30行 |
| 多语言支持i18n(v2) | — | Token层新增字体变量，文案走i18n key映射 | CSS 2行 + i18n文件 |
| 移动端原生App(v2) | — | 响应式第3档(<768px)已是单栏，需重构为React Native | 完整新项目 |

### 22.3 不可破坏的契约

以下设计决策不可在v1/v2中随意推翻（否则需整体重设计）：

- **Dark Mode OLED 基调**：所有新增界面必须以 `--bg-page: #0d0d0f` 为根底色
- **三栏黄金比例**：新增面板优先放入右侧360px栏，不改变280:flex:360比例
- **CSS变量引用强制**：新增样式禁止硬编码颜色/间距/圆角，必须引用第0节Token
- **组件变体追加不改签**：新增按钮variant只能追加第5个，不能修改前4个的样式语义
- **状态流转只追加**：新增状态只能追加在"归档态"之后，不能插入中间改变已有编号
- **Nginx为唯一入口**：所有HTTP/WS流量必须经过Nginx，不可在v2绕过直接暴露服务端口
