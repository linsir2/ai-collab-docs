---

## 1. 测试范围

### IN SCOPE（纳入测试）

| 模块 | 功能点 | 依据文档 |
|------|--------|---------|
| 五不可变底线 | AI零直改、人类主权、立意锚、状态锁止、自动采纳可撤回 | PRD 二 |
| 状态机 | 5状态 × 8条transition × 5角色 = 40格权限矩阵 | PRD 六、design_report Round2 |
| 模块解耦 | 6 BC物理隔离、依赖方向、contracts纯净性、事件总线、数据所有权 | ga.md §架构约定 |
| 数字阈值 | 14项已量化（信任分≥80、申诉<7、Token 80%等）+ 9项未量化待试探 | PRD + UI design + design_report |
| 双轨AI | personalAI vs docAI隔离、公私记忆边界 | PRD 模块二、ADR-002 |
| 轻量/重型模式 | 状态图差异、模式锁定 | PRD 六 |
| 设计内一致性 | model_repo vs design_report vs PRD 交叉校验 | 全部设计文档 |

### OUT OF SCOPE（明确排除）

| 排除项 | 原因 |
|--------|------|
| 第三方SDK内部逻辑（Yjs、DashScope、PostgreSQL、Redis） | 信任厂商 |
| 性能调优后的极限压测 | 另立项 |
| 渐进信任自动采纳（v1功能） | MVP暂缓，见 ga.md |
| 向量化项目记忆 + 语义检索（v1功能） | MVP暂缓 |
| 轻量创作模式全链路（v1功能） | MVP先验证重型锻造核心价值 |
| 离线协同增强UX | v2 |
| 模板中心 + 企业管理面板 | v2 |

### 假设条件

1. 测试环境与 MVP 部署环境配置一致（或差异已记录）
2. 测试数据已脱敏，不含真实用户信息
3. LLM 返回值在每个测试 session 内稳定（embedding 模型版本锁定）

## 4. 风险识别

| 风险 | 概率 | 影响 | 预案 |
|------|------|------|------|
| 未量化阈值（P-001~P-009） | — | — | ✅ 已全部决议（见 test-strategy.md §5），P-001余弦0.8/P-002信任分MVP=50/P-003~P-009均有明确结论 |
| model_repo 数据不全（P-008） | 中 | 中 | 从 design_report Round 2~6 回填数据 |
| Yjs/Redis/PostgreSQL 环境不稳定 | 中 | 高 | 准备容器化一键部署脚本，确保可快速重建 |
| LLM embedding 模型切换导致余弦相似度变化 | 中 | 中 | 阈值试探时记录所用模型版本，模型升级时重跑样例集 |
| 设计变更 | 低 | 高 | 每次设计变更后同步更新测试文档，版本号递增 |
