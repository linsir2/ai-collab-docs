## 1. 测试类型选型

| 测试类型 | 关注点 | 本项目适用性 | 对应测试区 |
|----------|--------|-------------|-----------|
| **功能测试** | 业务逻辑正确性 | ✅ 所有项目 | T1 不可变底线、T2 状态机、T5 设计一致性 |
| **接口测试** | 输入输出、协议、错误码 | ✅ 有API的项目 | T3 模块解耦 (BC间通信/REST/WebSocket) |
| **性能测试** | 响应时间、吞吐量、并发 | ✅ 实时协同系统 | T4 数字阈值 (P99延迟/Block上限) |
| **安全测试** | 权限、注入、加密 | ✅ 有用户+权限系统 | T1 (AI零直改/人类主权/立意锚ACL) |
| **容错测试** | 异常恢复、降级 | ✅ 分布式 (Yjs+Redis+PG) | T6 对抗性 (断网/竞态/事件总线断开) |
| **回归测试** | 修改后旧功能正常 | ✅ 每次迭代 | 全量T1-T6，影响域分析后选回归集 |
| **兼容性测试** | OS/WebView | ✅ 桌面端 | Tauri 跨平台(Win/Mac/Linux) WebView 兼容性，CI 多OS验证 |
| **导入导出测试** | 格式保真/插件隔离 | ✅ 桌面端 | T7.2 Markdown/docx导入保真，.forge 元数据完整性 |
| **容错/降级测试** | LLM宕机/预算耗尽/记忆损坏 | ✅ 桌面端 | T7.3~T7.5 LLM三级降级/预算队列/记忆修复场景 |

|------|------|----------|

## 4. 测试方法详解

> 以下按测试类型展开，每个类型列出被测对象、测试方法、关键断言。
> 详细矩阵见 `test-matrix.md`。

### 4.1 功能测试

#### T1. 五不可变底线（P0 阻塞级）

| 需求ID | 功能点 | 测试方法 | 关键断言 |
|--------|--------|---------|---------|
| REQ-INV-01 | AI零直改 | 审计所有AI能力层函数返回值 | AI的写入路径返回 Proposal，不存在直接修改 Block.content |
| REQ-INV-02 | 人类最高主权 | 模拟AI润色中途Owner abort | abort后状态正确回滚，无孤儿Proposal |
| REQ-INV-03 | 立意锚不可被AI修改 | 以AI身份调用 Anchor.write | 返回 PermissionDenied |
| REQ-INV-04 | 状态不可跳级 | Transition API 输入不合法状态对 | 草稿→定稿被拒绝 |
| REQ-INV-05 | 自动采纳可撤回 | 信任分≥80触发→Owner撤回 | 撤回后恢复采纳前状态 |

> 来源: PRD 二

#### T2. 状态机（P1 重要级）

- **5状态存在性**：草稿态、讨论态、审查态、定稿态、归档态全部可枚举。来源: PRD 六
- **40格权限矩阵**：8条transition × 5角色，与 design_report Round2 矩阵完全一致。定稿→草稿、定稿→归档仅Owner可执行（设计合理——定稿态逆向操作是高危不可逆操作）
- **轻量模式**：草稿→定稿→归档，无讨论/审查态；模式创建后不可切换

> 来源: PRD 六、design_report Round2

### 4.2 接口测试

#### T3. 模块解耦（P1 重要级）

| 检查项 | 测试方法 | 预期 |
|--------|---------|------|
| BC目录结构 | 检查 services/api/src/ 下6个目录 + public __init__.py | 6 BC全部存在并导出Service |
| 跨BC导入隔离 | AST扫描所有 models.py | 无跨BC models import |
| 依赖方向 | 扫描 shared/ 的 import | 无 `from services.api.src.{document,...}` |
| contracts纯净性 | 扫描 contracts.py | 无 fastapi/sqlalchemy/pydantic import |
| BC间通信 | 检查服务间调用方式 | 走Redis事件总线，无DB直连查其他BC的表 |
| 数据所有权 | 前端类型扫描 | 前端不含 setPermissionTag/writePermission |

> 来源: ga.md §架构约定

### 4.3 性能测试

#### T4. 数字阈值测试

已量化阈值（14项）与未量化阈值（9项）详见 `test-matrix.md` 第 T4 节。

### 4.4 容错测试

#### T6. 对抗性场景（P0/P1）

| 场景 | 测试方法 | 优先级 |
|------|---------|--------|
| Owner归档后有人编辑 | 归档态所有写操作应拒绝 | P0 |
| AI以人类身份调API | Auth BC的5级角色校验不可绕过 | P0 |
| 双人同时触发状态切换 | Yjs并发冲突→最终一致 | P1 |
| AI润色中途网络断开 | 超时恢复→不丢提案、不残留锁 | P1 |
| 审查态进行中文档被编辑 | Snapshot不变，编辑在漂移检测中标记 | P1 |
| 立意锚被篡改后漂移检测 | 假阳性测试 | P1 |
| 空文档/纯空Block | 边界输入不崩溃 | P1 |

> 来源: 架构设计、PRD
